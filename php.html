5.4 Sessions & cookies

    Un cookie est souvent utilisé pour identifier un utilisateur. Un cookie est un petit fichier que le serveur intègre sur l'ordinateur de l'utilisateur. Chaque fois que le même ordinateur demande une page avec un navigateur, il enverra également le cookie. Avec PHP, vous pouvez à la fois créer et récupérer des valeurs de cookies.
    Les sessions permet de stocker des données différentes pour chaque utilisateur en utilisant un identifiant de session unique, ils sont principalement utilisés dans les systèmes d'authentification, on utilise généralement $_SESSION pour récupérer toutes les données d'une session, chaque session en php commence par session_start() et finit par la fermeture du navigateur ou bien en utilisant d'autres méthodes de cookies.

<?php $defaultLng = 'en';
setcookie("lng", $defaultLng);
setcookie("lng", $defaultLng, time()+10); // cookie avec 10 secondes comme durée de vie echo $_COOKIE["lng"]; session_start();
$_SESSION["email"] = "user@email.com";
$_SESSION["login_date"] = "2022-12-17"; var_dump($_SESSION); ?>

5.5 POO

La notion de la programmation orienté objet a été introduite en php depuis la version 5 (2004), La programmation orientée objet se base sur un concept fondamental qui est que tout élément dans un script est un objet ou va pouvoir être considéré comme un objet. chaque objet appartient à une famille d'objets (classe), chaque classe dispose d'attributs et de méthodes qui lui sont spécifiques, qui peuvent être partagées par la suite avec d'autres classes via l'héritage.

<?php class Car {
public $name;
public $manufacture;
public $date;
public $serial_number;
public $max_speed;
function __construct($name,$manufacture,$date,$serial_number,$max_speed ){
$this->name = $name;
$this->manufacture = $manufacture;
$this->date = $date;
$this->serial_number = $serial_number;
$this->max_speed = $max_speed; }
}
$car = new Car("clio 4","renault","2022-12-10","0254687-115-06",230);
var_dump($car);
// le mot extends est utilisé pour l'héritage class Car extends Vehicle{ public $type;
}
?>

5.5.0 Héritage

L'héritage est l'un des quatre piliers de la programmation orientée objet (POO). L'héritage est le phénomène par lequel une classe enfant peut hériter de toutes les propriétés et caractéristiques de la classe mère. Vous pouvez comprendre cela avec un exemple simple et réel. Prenons l'exemple des êtres humains. Vous héritez des traits caractéristiques de la classe « Humains », tels que marcher, s'asseoir, courir, manger, etc. La classe « Humains » hérite de la classe « Mammifère », ces traits caractéristiques - faisant de la classe « Humain » une classe dérivée de 'Mammifère'. De plus, la classe "Mammifère" tire ses traits caractéristiques d'une autre classe encore - "Animal". Cela fait de la classe « Mammifère » une classe dérivée de la classe « Animal » ; faisant également de la classe "Animal" une classe de base.

5.5.1 Interface

Une interface PHP définit un contrat qu'une classe doit remplir. Si une classe PHP est un modèle pour les objets, une interface est un modèle pour les classes. On peut s'attendre à ce que toute classe implémentant une interface donnée ait le même comportement en termes de ce qui peut être appelé, comment elle peut être appelée et ce qui sera retourné.

5.5.2 polymorphisme

Le polymorphisme décrit un exemple dans la programmation orientée objet où les méthodes de différentes classes qui font des choses similaires devraient avoir un nom similaire. Le polymorphisme est essentiellement un modèle OOP qui permet à de nombreuses classes avec différentes fonctionnalités d'exécuter ou de partager une interface commune. L'utilité du polymorphisme est que le code écrit dans différentes classes n'a aucun effet sur la classe à laquelle il appartient car ils sont utilisés de la même manière 

                    <?php 

                    interface Greeting {
                        public function greet();
                     }
                     class French implements Greeting {
                        
                        public function greet(){
                           echo "Bonjour";
                        }
                     }
                     class English implements Greeting {
                        public function greet(){
                            echo "Hello";
                         }
                     }
                     class German implements Greeting {
                        public function greet(){
                            echo "hallo";
                         }
                     }
                     
                    ?>
